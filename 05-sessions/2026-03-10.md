# 2026-03-10

## AI Boss 자동화 — 데몬 + 스케줄러 등록

### 배경
AI Boss 코어 로직(체크인, 응답, 페르소나)은 구현 완료 상태였으나 자동 트리거가 없어 수동 실행만 가능했음.
- 체크인: scheduler에 미등록
- 응답: bridge가 cc-boss로 라우팅하지만 소비 프로세스 부재

### 구현

1. **`scripts/ai_boss/boss_daemon.py`** — MsgBus polling 데몬 (신규, ~110줄)
   - MsgBus `cc-boss` recipient를 3초 간격으로 polling
   - 메시지 수신 → `responder.respond_to_message()` 호출 → Telegram 응답 전송
   - signal 핸들링 (SIGINT/SIGTERM), RotatingFileHandler (2MB, 3백업)
   - 로그: `storage/logs/boss-daemon.log`

2. **`scripts/scheduler.py`** — 체크인 seed 작업 3개 추가
   - `ai-boss-morning`: 매일 09:00, bridge 서비스 의존
   - `ai-boss-evening`: 매일 18:00, bridge 서비스 의존
   - `ai-boss-weekly`: 금요일 17:00, bridge 의존 + 144h 최소 간격

3. **`scripts/daemon.sh`** — boss 서비스 등록
   - tmux 세션 `cc-boss-daemon`으로 관리
   - start/stop/restart/status/logs 전부 지원

4. **`scripts/healthcheck.sh`** — boss 자동 재시작 커버
   - 60초마다 boss 프로세스 확인 → 죽으면 자동 재시작

5. **PATH 버그 수정** — `responder.py`, `checkin.py`
   - tmux/launchd 환경에서 `claude` CLI를 못 찾는 문제
   - `~/.local/bin`을 PATH에 자동 추가하는 보정 코드 삽입

### 아키텍처

```
[체크인 흐름]
scheduler.py (cron) → checkin.py morning/evening/weekly
  → Claude CLI (haiku) → 응답 생성
  → sender.py → MsgBus → bridge → Telegram boss 그룹

[응답 흐름]
상엽 (Telegram boss 그룹)
  → bridge → MsgBus (recipient="cc-boss")
  → boss_daemon.py (3초 polling)
      → responder.py (Claude CLI haiku)
      → sender.py → MsgBus → bridge → Telegram boss 그룹
```

### 검증
- `daemon.sh start/stop/restart boss` — 정상 동작
- `daemon.sh status` — boss 서비스 표시 확인
- scheduler seed — 3개 작업 등록 확인
- PATH 수정 후 `FileNotFoundError` 해결 확인
- 중첩 세션 에러는 Claude Code 안에서 테스트한 한계 (독립 실행 시 정상)
