## 3-Tier 아키텍처 구현

### 핵심 변경
- **planner 세션**: cc-planner 1개. idea → spec 전담. worker와 분리.
- **feature 명령**: `feature "제목" "설명"` → idea 생성 + planner 할당 + 코봉 알림
- **spec 승인 흐름**: spec_ready → `approve cc-planner` → queued → scheduler가 worker 배정
- **reject planner**: `reject cc-planner "피드백"` → planner에 수정 지시

### 새 파일
- `subagent_spawn.py`: build_planner_prompt(), build_worker_prompt(), assign_to_planner()
- DB: session_pool.session_type='planner'|'worker', work_queue.stage에 'idea'/'spec' 추가

### 흐름
```
feature "여행 기능" → idea(#7) → planner 할당 → spec 작성 → spec_ready
  → 코봉 알림 → 상엽 승인 → queued → scheduler → worker 배정 → designing
```
