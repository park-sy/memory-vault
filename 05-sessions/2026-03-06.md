# Session: 2026-03-06

## 서비스 데몬 관리 인프라 구축

### 작업 요약

bridge/factory 데몬을 수동 tmux 관리에서 자동화된 인프라로 전환.

### 산출물

| 파일 | 작업 |
|------|------|
| `scripts/daemon.sh` | **신규** — 통합 서비스 관리 CLI (start/stop/restart/status/logs/install/uninstall) |
| `scripts/healthcheck.sh` | **신규** — launchd 60초 주기 liveness 체크 + 자동 재시작 |
| `scripts/telegram_bridge.py` | file logging 추가 (RotatingFileHandler → `storage/logs/bridge.log`) |
| `scripts/create-domain-channel.sh` | bridge 재시작 로직을 `daemon.sh restart bridge`로 교체 |
| `CLAUDE.md` | 서비스 관리 섹션 추가 |
| `communication-system.md` | daemon management 참조 추가 |

### 기술 결정

- **tmux 기반 유지**: vault-ui, pool.sh가 `tmux has-session`으로 상태 판단하므로 launchd 직접 프로세스 관리 불가. launchd는 healthcheck 역할만.
- **bash 3.2 호환**: macOS 기본 bash가 3.2라 `declare -A` (associative array) 사용 불가. case문 기반으로 서비스 정의.

### 검증 결과

- `daemon.sh status` — 상태 표시 정상
- stop/start 사이클 정상
- healthcheck 자동 재시작 확인 (bridge kill → 감지 → 재시작 → Telegram 알림)
- launchd plist 등록 완료 (리부팅 후 자동 복구)
