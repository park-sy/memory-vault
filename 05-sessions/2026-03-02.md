## Feature Factory E2E Dogfooding — factory-monitor 첫 생산

Feature Factory 데몬을 실제로 띄워서 `factory-monitor` feature를 자동 생산한 첫 e2e 검증.
"컴파일러가 자기 자신을 컴파일" — Feature Factory로 Feature Factory 모니터를 만듦.

### 실행 결과

| 단계 | 역할 | 소요 | 승인 게이트 | 산출물 |
|------|------|------|------------|--------|

**총 소요: 57분** (타이밍 버그 포함), **실제 작업: ~26분**

### 발견된 버그 2건 + 수정

1. **whos-life `_exec_write` 버그** — UPDATE 문에 `fetchall()` 호출 → `ResourceClosedError`
   - 수정: `db/database.py`에 `result.returns_rows` 분기 추가

2. **pipeline_manager venv 경로 문제** — `sys.executable`(시스템 Python)으로 whos-life CLI 호출 → sqlalchemy 없음
   - 수정: `config.py`에 `WHOS_LIFE_PYTHON` 추가, `pipeline_manager.py`에서 사용

3. **워커 startup 타이밍 버그** (핵심!) — `worker_manager.py`의 `time.sleep(3)` → Claude Code 시작 전에 메시지 inject → 프롬프트 화면 표시되나 실제 입력으로 안 받아들임
   - 수정: `time.sleep(3)` → `time.sleep(15)`
   - 매 단계마다 수동 재전송 필요했음 (데몬 재시작 안 해서 old 코드 적용)

### 핵심 교훈

- **supervision=OFF 자율 실행 성공**: 워커가 task 수신→작업→알림까지 자율 완수
- **타이밍 버그가 가장 큰 장애물**: Claude Code startup에 3초는 부족. 15초로 수정.
- **MsgBus callback 승인 정상 작동**: Telegram 없이 CLI로 직접 승인 가능
- **파이프라인 자동 전이 정상**: idea→spec→queued→designing→testing→stable→coding→done
- **역할 전환 정상**: coder→planner→coder→qa→coder 자동 전환
- **Production report 자동 생성** 확인
