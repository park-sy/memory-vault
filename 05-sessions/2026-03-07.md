## ai-ops 피처 중복 정리 + token-dashboard → token-monitor 통합

### 배경
ai-ops 7개 feature에 유틸리티 함수가 복사-붙여넣기로 퍼져있었고, token-monitor와 token-dashboard가 같은 Anthropic rate limit API를 독립적으로 호출하는 중복이 있었음.

### Part A: `_shared.py` 유틸리티 추출
- **`features/ai-ops/_shared.py`** 신규 (~305줄)
  - `KST`, `read_external_db()`, `utc_to_kst()`, `format_time_short()`, `format_time_full()`, `format_reset_time()`
  - `format_model_name()` — claude-/gemini- 접두사 처리 통합 (3벌 → 1)
  - `run_openclaw()` — None 반환 패턴으로 통일 (agent-manager ERROR 문자열 → None)
  - `get_tmux_sessions()`, `tmux_session_has_process()`, `scan_tmux_cc_sessions()`
  - `read_oauth_token()`, `parse_rate_limit_data()`, `fetch_anthropic_rate_limits()`
  - `_WINDOW_LABELS` 6키 (token-dashboard의 확장 버전 채택)
- **소비자 12개 파일 마이그레이션** — 각 파일에서 중복 함수 제거 + `from .. import _shared` 교체

### Part B: token-dashboard → token-monitor 통합
- `token-monitor/render.py` 탭 구조 변경: [모니터링 | 사용량 분석]
- `token-monitor/scanner.py`, `token-monitor/models.py` — token-dashboard에서 이동
- `token-monitor/service.py` — dashboard service 함수 병합 (`scan_and_persist`, `get_dashboard_data`, `format_tokens` 등)
- `token-dashboard/` 디렉토리 전체 삭제
- 테스트 import 경로 수정 (`conftest.py` + `test_token_scanner.py` + `test_token_service.py`)
- `_shared` 모듈을 conftest에 등록하는 `_register_shared_module()` 추가
- Python 3.9 호환: `str | None` → `Optional[str]` (models.py)
