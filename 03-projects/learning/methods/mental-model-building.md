# 멘탈 모델 구축 (Mental Model Building)

## 개요

세상이 작동하는 방식에 대한 내부 표상(멘탈 모델)을 의식적으로 구축하고 정제하는 방법.
"사고의 도구"를 늘리면 같은 문제를 다양한 렌즈로 볼 수 있다.

## 원리

- 스키마 이론: 새 정보는 기존 스키마에 연결될 때 이해/기억됨
- 전이 학습: 한 분야의 모델이 다른 분야에 적용 가능 (유비추론)
- 격자형 사고(lattice of mental models): 여러 모델의 교차점에서 통찰 발생

## 절차

1. 핵심 모델 식별 (파레토, 피드백 루프, 제약 이론 등)
2. 모델의 핵심 구조를 자신의 말로 정리
3. 자신의 경험에서 모델이 적용되는 사례 3개 이상 찾기
4. 모델이 적용되지 않는 한계/경계 조건 파악
5. 다른 모델과의 관계/충돌 매핑

## 변형

- **모델 충돌 분석**: 두 모델이 상반된 예측을 할 때 어떤 것이 맞는지 판별
- **경험 기반 모델링**: 상엽 자신의 경험에서 귀납적으로 모델 추출
- **AI 모델 생성**: LLM에게 "이 현상을 설명하는 멘탈 모델 3개 제안해줘" 요청

## When to Use

- 새 분야 전체 조망이 필요할 때
- 의사결정의 프레임워크가 필요할 때
- [[case-study-analysis]]에서 추출한 패턴을 일반화할 때

## When NOT to Use

- 구체적 사실/절차 암기 (과도한 추상화)
- 이미 잘 작동하는 직관이 있는 영역
