# Learning — 도메인 컨텍스트

AI를 활용한 학습 최적화 도메인팀. 코어팀에 전달할 최소 컨텍스트.

## 프로젝트 요약

다양한 학습 방법을 탐색 → 상엽에게 적용/실험 → 데이터 기반으로 최적 방법 수렴.
소프트웨어 프로젝트가 아닌 **실험/최적화** 유형 도메인팀.

## 학습 프로필 — 상엽

### 강점
- 비유를 통한 이해가 빠름 (특히 자기 경험에 기반한 비유)
- 시스템/구조적 사고 (개발자 배경)
- "왜?"에 대한 강한 호기심 — 원리를 이해해야 적용 가능
- 반복보다 패턴 추출을 선호

### 약점
- 단순 암기형 학습에 약함
- 흥미가 없으면 집중도 급락
- 초기 진입 장벽이 높은 주제에 대한 인내심 낮음

### 선호 모달리티
- 대화형 > 독서형
- 시각화 > 텍스트
- 실습 > 이론 선행
- 짧은 반복 > 긴 세션

### 비유 재료 — 비개발 경험

| 영역 | 경험 | 비유 활용 포인트 |
|------|------|-----------------|
| 게임 | LoL 미드 메이지 (애니비아, 빅토르) | 포지셔닝, 파워 스파이크, 팀파이트 타이밍, 라인전 트레이드, 스케일링 vs 초반 압박, 시야 장악 |
| 운동 | 수영 | 반복 훈련으로 폼 체화, 호흡 리듬, 체력 관리 |
| 학문 | 경제학과 전공 (기억 희미) | 수요/공급, 기회비용, 한계효용, 인센티브 구조 — 기초 개념은 비유로 활용 가능 |
| 학회 | SRS 투자학회 리서치팀 (기억 희미) | 리서치 프로세스, 투자 분석 프레임 |
| 개발 | memory-vault, whos-life 인프라 구축 | 아키텍처, 파이프라인, 자동화, 트레이드오프 (가장 풍부) |

### 학습 관심 분야
- 경제/투자 (과거 배경 살리기)
- 비즈니스/창업

## 핵심 가설

> **개인 경험 기반 비유(Contextual Analogy) + AI 선순환 구조**가 가장 효과적인 학습 방법이다.

memory-vault에 상엽의 경험이 축적됨 → AI가 그 경험을 기반으로 비유 생성 → 이해도 상승 → 새 경험 축적 → 비유 정확도 상승 (선순환).

이 가설은 아직 **실험 전**이며, 체계적 검증이 필요하다.
