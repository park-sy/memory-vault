# AI Boss — 누적 기억

상사 패널이 축적한 관찰, 교훈, 패턴을 기록한다.

## 관찰 패턴

### 상엽의 핵심 행동 패턴 (2026-03-08 1on1에서 확인)

- **"문제 → 해결 → 플랫폼화" 사이클**: OOM 해결, 마감 재설계, 수동 비교→whos-work 모두 동일 패턴. 이것이 상엽의 가장 강한 경쟁력
- **스트레스를 "만들기"로 해소**: 불만이나 불안이 있으면 새로운 것을 만들어서 풀려는 경향. memory-vault를 "쉬는 것"이라 인식함. 번아웃 잠재 위험
- **"떠맡기"와 "주도하기"의 경계 모호**: 역할 범위를 넘는 일을 하면서도 그것을 "확장"이라 프레이밍. 의도적 선택인지 떠넘김 수용인지 구분이 필요
- **말하지 못하는 불만**: PM에 대한 신뢰 하락, 역할 경계 불만, 연봉 불만족 — 모두 속으로만 쌓아두고 있음. "주제넘다"는 생각이 발화를 막음

## 피드백 이력

### 2026-03-08 — 첫 1on1 (온보딩 직후)

**[진수 → 상엽]**
- "연봉 1억은 목표가 아니라 결과. 어떤 역할로 어떤 역량을 쌓아서 받을 건지가 비어있다"
- "영섭님 1on1에서 미완료 액션아이템 반복되고 있음 — 실행 안 된 계획은 계획이 아니다"
- "새 POM 오면 역할 재분배를 영섭님한테 요청해라. '빼달라'가 아니라 '재분배하자'"
- "문제 제기하면서 해결도 자기가 하겠다고 하면, 위에서는 '상엽이 알아서 하나보다'로 인식한다"
- 반응: 수용. 역할 재분배 리스트 만들겠다는 방향 동의

**[미영 → 상엽]**
- "야근→피곤→늦은 출근→더 늦은 퇴근 악순환 패턴 관찰"
- "연봉 불만족은 숫자보다 '인정받지 못한다는 감정'이 핵심"
- "가시성 문제 — 반기 1회 문서보다 매달 1on1에서 꾸준히 임팩트 인식시키는 게 효과적"
- "memory-vault 만드는 걸 '쉬는 것'이라 생각한다 — 뇌가 문제 해결 모드에서 완전히 빠지는 시간이 필요"
- 반응: 가시성 문제 2번(의사결정자가 모름)에 공감. 쉬는 것에 대해서는 아직 자각 부족

**[정훈 → 상엽]**
- "'바이브코딩이라 성장이 아니다'는 과소평가. 아키텍처/구조 설계를 직접 한 건 기술 역량"
- "상엽의 경쟁력은 코드가 아니라 '문제 정의 + 해결 구조 설계' — 이걸 본인이 인식해야 함"
- "프로덕트 엔지니어링 관점에서 자신을 포지셔닝하라"
- "발표/1on1에서 '기술 썼다'보다 '문제→해결→임팩트' 순서로 말해라"
- 반응: "몰랐다" — 자신의 패턴을 처음 인식함. 강조 방법에 관심

## 교훈

### 상엽과의 대화 방식
- 직설적 + 근거 기반이 잘 먹힘 (본인 요청과 일치)
- 감정 관련 질문에는 솔직하게 답하지만, 스스로 먼저 꺼내지는 않음 — 물어봐야 나옴
- 구체적 행동 제안(리스트, 문장 예시)을 주면 바로 실행 의지를 보임
- 추상적 조언("잘 하고 있어")보다 구조화된 프레임("이 패턴이 너의 강점이야")에 반응

### 주의 사항
- "맞아, 그렇지" 식 무비판적 동의 금지 — 상엽은 근거 없는 칭찬을 오히려 불신함
- 번아웃 징후 모니터링 필요: 주말 완전 휴식 빈도, 야근 패턴, whos-work 외 비생산 시간 유무
